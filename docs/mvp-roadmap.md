# MVP Roadmap

## 阶段 0：仓库和定位

目标：

- 建立 GitHub 可发布的基础仓库
- 明确产品定位、目标用户和差异化

当前内容：

- landing page
- README
- 产品定位文档
- 路线图文档

## 阶段 1：最小可用版本

目标：

- 让用户填写基础信息
- 生成第一版学习与求职路线

建议功能：

1. 用户 onboarding 表单
2. 路线生成结果页
3. 每周任务卡片
4. 聊天入口

## 阶段 2：执行闭环

目标：

- 让产品不只是“给建议”，而是真的推动执行

建议功能：

1. 任务完成打卡
2. 周报复盘
3. 学习阻塞点记录
4. 下一周自动调计划

## 阶段 3：求职深化

目标：

- 让求职相关功能形成完整链路

建议功能：

1. 岗位方向匹配
2. 简历优化建议
3. 模拟面试问答
4. 面试复盘记录
5. 投递节奏面板

## 阶段 4：AI 资料系统

目标：

- 让 agent 真正具备“整理资料”和“按资料答疑”的能力

建议功能：

1. 上传 PDF/笔记
2. 检索资料内容
3. 基于资料回答问题
4. 自动生成学习摘要和题单

## 暂时不要优先做的内容

- 复杂支付系统
- 多角色后台
- 实时语音教学
- 泛化到所有专业
- 过早追求大而全的社区功能
