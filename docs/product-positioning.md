# Product Positioning

## 一句话定位

一个面向计算机大学生的 AI 学习与求职共驾系统，帮助用户从迷茫、焦虑和信息过载中走出来，形成可执行、可追踪、可沉淀的成长路径。

## 核心用户

- 大二到大四的计算机相关学生
- 对前端、后端、算法、测试、数据、客户端等方向有求职需求的人
- 有一定学习意愿，但缺少方法、节奏和外部反馈的人

## 用户真实痛点

1. 不知道自己适合什么岗位
2. 不知道当前能力差距在哪里
3. 收藏了很多资源，但无法建立优先级
4. 学习和求职脱节，导致投入了很多时间却没有转化为竞争力
5. 缺少持续陪跑和阶段反馈，容易拖延或半途放弃

## 培训机构的典型问题

1. 方案标准化，难以真正贴合个人背景
2. 关注成交和转化，容易利用求职焦虑
3. 输出偏课程而不是可验证成果
4. 服务周期有限，结束后用户容易再次失去方向

## 我们的竞争优势

1. 长期记忆
agent 会逐步记住用户的目标岗位、学习状态、项目经历、简历版本和面试反馈。

2. 行动闭环
不是只告诉用户“学什么”，而是拆成今天、本周、本月的任务，并在执行后继续复盘。

3. 学习与求职一体化
很多工具只做资料推荐，很多服务只做简历和面试，我们把二者打通。

4. 以 GitHub 和项目输出为导向
对计算机学生来说，真正的竞争力往往来自项目、代码、文档和表达能力，而不是单纯“看完了多少课”。

5. 更低成本、更高频
相比高价培训和低频咨询，这个产品更适合日常使用和持续陪跑。

## 不应该做成什么

- 不要做成“AI 版网课超市”
- 不要只做一个通用聊天机器人
- 不要把产品重点放在花哨功能而不是执行闭环
- 不要一开始就尝试覆盖所有专业和所有年龄层

## 最佳切入点

第一阶段建议只服务一类人：

- 计算机专业学生
- 目标是找技术岗实习或校招
- 希望通过学习路线 + 项目输出 + 求职节奏获得实质进展

这样做的好处是：

- 用户需求更集中
- 资料体系更容易沉淀
- agent 的提示词和规划逻辑更容易做深
- GitHub 项目展示也更聚焦

## 产品信任感来源

1. 透明解释为什么推荐这条路线
2. 清楚指出能力差距，不夸大承诺
3. 给出可验证的学习成果
4. 允许用户看到自己的进步轨迹

## 成功指标

- 用户是否能持续使用一周以上
- 用户是否完成了更多高质量任务
- 用户是否产出了更多项目、简历版本和面试复盘
- 用户是否更清楚自己的方向，而不是更焦虑
